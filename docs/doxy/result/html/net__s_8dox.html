Detailed Description

Technical description of net_s as a full neural network container.

Author
Oscar Sotomayor (Titux)

This document describes net_s strictly as a structural descriptor of the network. It defines layers, neurons, buffers (bff), input/output pointers, and connectivity information. net_s does not execute computations by itself; all execution logic is implemented externally.

No learning, optimization, or training concepts are included here.

Concept

net_s represents the full network container. Its purpose is to store:

  • Input references
  • Layer definitions (first layer, optional middle layers)
  • Neuron arrays per layer
  • Intermediate buffers (bff) for inter-layer communication
  • Wiring descriptors (wiring)
  • Output pointers

net_s is purely structural. External code iterates over layers and neurons, reads inputs, applies operations (from neuron_s), and writes to outputs.

Structural View

The structure can be visualized as:

External Information Flow

Information flow is defined outside the structure. External responsibilities:

  • Iterating over layers
  • Reading data from inputs
  • Using bff for intermediate buffers (if middle layers exist)
  • Evaluating each neuron_s instance
  • Writing results to out pointers

net_s only provides the framework to organize these operations.

Referential Connectivity

Inputs and outputs are accessed via pointers, enabling:

  • Shared data sources
  • Direct neuron-to-neuron connections through bff
  • Integration with external variables without copying

Buffers and Wiring

  • bff: optional buffers connecting layers. Only present if net_s.layers > 1. Can handle multiple arrays per neuron-to-neuron connection.
  • wiring: descriptor of connectivity; it does **not** transport data.
Warning
  • Accessing bff when there are no middle layers is undefined.
  • wiring should never be modified directly outside network construction functions.
  • All internal neuron in/out pointers are only valid within the external evaluation loop.
